Acerca de Bootstrap

¿Qué es Bootstrap?


Bootstrap es un framework orientado al desarrollo web centrado en el enmaquetado HTML y diseño CSS. Una de sus características es lograr un diseño adaptativo para los diferentes dispositivos que pueden conectar con nuestra aplicación: PC, Tablet, Smartphone, etc. Esta técnica de diseño y desarrollo se conoce como Responsive Design o Diseño Adaptativo.

Bootstrap es el elemento que se ve primero en nuestro sistema web. Después irían todos los demás. Este framework sirve sobre todo para poder adaptar el diseño sobre una aplicación en varios dispositivos sin tener que adaptar cada uno de forma independiente.

Cuando se comienza a utilizar Bootstrap, se incorporan diferentes plantillas para los documentos que pueden utilizarse en las aplicaciones web resultantes. El diseñador está condicionado a un diseño. Pero lo mejor es que es mucho más rápido. No se tiene que empezar de 0.

Ventajas


Facilidad de uso
Este Framework evita el tener que preocuparse por las media queries y los porcentajes en los archivos CSS para hacer una web Responsive, facilitando la programación del sitio.

Simplicidad
Se basa en la simplicidad de sus interfaces, una tendencia del mercado, en las que se tiende a diseño plano, botones grandes y buena organización, para facilitar la usabilidad en los dispositivos más pequeños, atendiendo a la Experiencia de Usuario.

Buena composición
Se puede diseñar una web jugando con sus elementos compuestos por diferentes combinaciones de HTML, CSS y Javascript.

Buena adaptación
Sus plantillas son de sencilla adaptación responsive, que permiten integrarse con librerías JavaScript, jQuery o GitHub. Facilita enormemente el diseño de interfaces y, además, incluye por defecto una plantilla bastante optimizada.

Amplia documentación
Cuenta con una documentación completa que te puede sacar de apuros rápidamente.

Mantanimiento propio
Cuenta con un mantenimiento y actualización realizados por Twitter: gran parte del trabajo interno ya está llevado a cabo por sus creadores.

Desventajas


Aprendizaje
Es necesario adaptarse a su forma de trabajo. Su curva de aprendizaje es liviana, pero es necesario comprender su estructura y nomenclatura. Por ello, se recomienda trabajar con Bootstrap desde el inicio de un proyecto, ya que si quieres incluir el framework en un trabajo ya iniciado algunos estilos podrían “romperse” y se tendría que ajustar a como se tenia en un principio, y eso puede ser un poco tedioso y molesto.

Diferentes versiones
Es complicado cambiar de versión si has realizado modificaciones profundas.

Demasiado pesado
Es un framework que no es ligero y, además, para algunas funcionalidades, será necesario tener que usar Javascript y jQuery.

Adaptación
Debes adaptar el diseño a un grid de 12 columnas, que se modifican según el dispositivo. Este es un grave problema, ya que Bootstrap trae por defecto anchos, márgenes y altos de línea. Esto conlleva a realizar cambios específicos dentro del archivo CSS del framework, para poder adaptar el framework a las necesidades del proyecto.

Ampliar componentes
Si se necesitan ampliar componentes no existentes en el archivo CSS, habrá que hacer dichas modificaciones y añadidos en el propio archivo para tener coherencia con el diseño del proyecto, mientras se mantiene el Responsive Design.

Limitaciones


No son pocas las desventajas de usar este framework. Es por ello que muchos desarrolladores se ven en la obligación de dejar de usar Bootstrap. Uno de ellos, Jeroen Ooms, Ingeniero Informático de Desarrollo e Infraestructura Web, describe cuáles son las limitaciones de este framework, por las cuales dejó de utilizar Bootstrap:

1.-Más de lo necesario
Es bastante raro que un desarrollador use todos los componentes disponibles de este frame. Lo que significa que habrá muchas líneas de código innecesarias en los archivos CSS y Javascript de Bootstrap, reduciendo el rendimiento del proyecto web.

2.-Predeterminado
Si trabajas para clientes, casi siempre hay que crear un diseño antes de que los desarrolladores se pongan a trabajar (en el mejor de los casos, tu también habrás estado involucrado como desarrollador). Este es otro problema con Bootstrap. Aparte de una serie de aspectos que se pueden personalizar, Bootstrap tiene un estilo predeterminado.

3.-Diseño y semántica
Bootstrap hace que sea casi imposible usar un patrón CSS de marcado (como BEM o SMACSS), porque los componentes de Bootstrap están vinculados al HTML con nombres de clase. Con esto también vinculas el estilo al marcado. Además del estilo, también vincula el comportamiento (javascript) de su sitio web a su HTML (nombres de clase y atributos de datos).

4.-El Sistema Grid
Un argumento que, a menudo, me escucho a mi mismo es "Necesitamos un sistema de rejilla". Por supuesto, existe un grid en Bootstrap y es casi imposible construir un sitio web responsive sin ello. Sin embargo, existen alternativas mucho mejores que lo que ofrece Bootstrap.

5.-Javascript
Todos los puntos anteriores estaban relacionados, sobre todo, con HTML y CSS. El Javascript de Bootstrap también tiene una serie de aspectos negativos. El javascript es un archivo grande, del que solo se pueden eliminar componentes a través del sitio web Bootstrap.

6.-jQuery
Aún mayor es el uso obligatorio de jQuery para la mayoría de los componentes de Bootstrap. Por lo tanto, usar el JavaScript Bootstrap nos reduce en flexibilidad y casi siempre resultará en recursos innecesarios.
